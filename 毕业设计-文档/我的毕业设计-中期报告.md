# 本科生毕业设计（论文）中期报告

## 题目
面向可扩展的大语言模型动态鲁棒测评方法研究

## 一、毕业设计（论文）主要研究内容、进展情况及取得成果

### 1. 研究内容
本课题围绕大语言模型测评中的鲁棒性与动态适应性问题展开，重点关注 `LLM-as-a-Judge` 在真实复杂场景下存在的随机性强、对输入扰动敏感、评测结果波动明显等问题。课题的总体目标是设计一种面向可扩展、动态且更具鲁棒性的大语言模型测评方法，使评测在不同输入扰动、不同任务形态以及后续新任务扩展中仍能保持较好的稳定性、一致性与可解释性。

在前期研究过程中，结合文献阅读、导师讨论和初步实验结果，当前工作逐步聚焦到 `query-specific rubric-based LLM-as-a-Judge` 这一更具操作性和研究价值的切入点。具体而言，本课题当前从以下几个层面推进：

1. 调研并梳理 `LLM-as-a-Judge` 现有方法及其鲁棒性问题，分析不同评测范式的特点与局限。
2. 研究在固定 `(question, response, rubric-list)` 条件下，Judge 单次打分的随机性及其对最终评测结果的影响。
3. 研究重复采样、criteria 聚合和 prompt 聚合对评测稳定性与和 Ground Truth 一致性的作用机制。
4. 初步探索自动生成 rubric 的方案，分析 rubric-list 本身变化对 query-level 评测结果的潜在影响。

总体来看，课题目前已经从“宏观讨论动态鲁棒测评框架”逐步收敛到“以 rubric-based 评测为核心切入点，系统分析其偏差、方差与聚合机制”的研究主线，并在此基础上向更动态的 rubric 生成与扩展评测方向推进。

### 2. 进展情况

#### （1）理论研究与文献调研
前期主要完成了 `LLM-as-a-Judge` 方向综述与代表论文的阅读，对单分制评估、pairwise 评估、rubric-based 评估等主流方法进行了分类整理，重点关注它们在稳定性、主观性、输入敏感性和适应性方面的差异。在此基础上，进一步检索并阅读了 `query-specific rubric-based` 方向的代表工作，并与导师讨论后认为，该技术通过为每个 query 构造更细粒度的评分标准，具备提升开放任务评测稳定性和可解释性的潜力，因此适合作为本课题的重点研究对象。

#### （2）实验框架初步搭建
目前已完成面向本课题的基础实验框架搭建，使用 `RubricHub_v1` 数据集中的前 100 条 query 作为主要实验对象，选取多个回答生成模型和多个 Judge 模型进行实验。项目中已经打通了数据读取、回答生成、Judge 调用、评分结果缓存、Ground Truth 对齐、统计分析与可视化的基本流程，并将主要过程模块化，为后续实验扩展提供了基础。

#### （3）重复采样与鲁棒性实验
在固定 prompt 与固定 rubric-list 的条件下，已完成对重复采样机制的初步实证分析。实验通过 bootstrap 方法模拟“采样 `n` 次并取均值”的过程，分别从 `criteria-level` 和 `query-level` 两个层面计算稳定性指标。结果表明，随着采样次数增加，Judge 输出波动显著下降，说明 LLM Judge 的单次评分不是稳定点值，而更适合被理解为某种分布样本。此外，实验还发现 `query-level` 的相对波动通常小于 `criteria-level`，为后续研究聚合效应提供了依据。

#### （4）重复采样与 Ground Truth 一致性实验
在稳定性实验基础上，进一步完成了 Judge 打分与 Ground Truth 的一致性分析，使用 Spearman 相关系数从 `criteria-level` 与 `query-level` 两个层面衡量 Judge 与参考标准的一致程度。实验结果表明，重复采样不仅能降低波动，通常还能提高与 Ground Truth 的排序一致性。这说明单次评测误差中有一部分确实是可通过平均削弱的随机噪声，而不是完全不可修正的系统偏差。

#### （5）输入敏感性实验与统一理论模型
围绕输入扰动问题，已完成多 prompt 变体条件下的实验分析。当前构造了多种 prompt variant，对比了单 prompt 与多 prompt aggregation 在稳定性和一致性上的差异。实验发现，多 prompt 条件下单次采样更分散，但在采样次数增大后，其一致性会逐渐追平甚至反超单 prompt。针对这一现象，已建立统一概率模型，将 Judge 评分表示为真实得分、prompt 层面的系统偏移和解码噪声三部分，并对两个关键现象进行了理论推导与证明：

1. 多 prompt 条件下的 `bias-variance crossover`；
2. `query-level` 相对 `SEM` 小于 `criteria-level` 的聚合效应。

这使当前工作开始从经验现象描述推进到带有统计解释的理论分析。

#### （6）高波动样本分析与影响因素探索
目前已初步开展 case study 和 factor analysis，对高波动样本、任务结构和若干数据特征进行检查。已有观察表明，价值判断型、边界模糊型、要求过宽型以及对抗性较强的问题更容易出现评分波动；同时，`rubric_list_len`、`response_words`、`gt_norm_score`、`gt_midness` 等变量与稳定性和一致性指标存在一定结构性关系。这部分工作帮助课题从“模型随机性”进一步推进到“任务结构与评分结构如何共同影响鲁棒性”的层面。

#### （7）自动生成 rubric 的预实验准备
在第八周开始，课题进一步向动态测评与可扩展方向延伸，重点讨论了自动生成 `query-specific rubric` 的方案。目前已完成 rubric 生成 prompt 和语义去重 prompt 的多轮调整，重点围绕 rubric 的原子性、可验证性、忠实性和非重叠性开展 prompt engineering，并结合 case study 对具体 question 的生成结果进行人工检查。当前已经形成较清晰的自动生成 rubric 预实验流程，包括重复生成 rubric-list、语义去重、构造 rubric 矩阵以及后续的 query-level 统计分析。

### 3. 取得成果
截至目前，课题已取得以下阶段性成果：

1. 完成了 `LLM-as-a-Judge` 及 `query-specific rubric-based` 方向的前期文献调研和方法梳理，明确了本课题的研究切入点。
2. 初步搭建了面向 rubric-based Judge 鲁棒性研究的实验框架，能够支持多模型、多次采样、Ground Truth 对齐和可视化分析。
3. 得到了较明确的实验结论：重复采样能够显著提升评测稳定性，并通常改善与 Ground Truth 的一致性。
4. 发现并验证了 `query-level` 相较 `criteria-level` 更稳这一重要现象，为后续研究 rubric 聚合机制奠定了基础。
5. 在输入扰动实验基础上建立了统一概率模型，并完成了两个核心现象的理论推导与证明，使当前工作具备了初步的理论支撑。
6. 完成了自动生成 rubric 方向的方案讨论与 prompt 初步优化，为后续从“固定 rubric”向“动态 rubric”扩展提供了实验准备。

从中期阶段来看，课题已经完成了从问题调研、实验框架搭建、核心实验验证到理论建模的前一半工作，整体进展与任务书要求基本一致，且研究主线较为清晰。

## 二、存在的问题和拟解决方案

### 1. 存在的问题

#### （1）课题总体范围较大，当前需要在“聚焦”与“扩展”之间保持平衡
任务书与开题报告中的目标覆盖动态测评、OOD 适应、输入扰动、任务迁移等较大范围，而当前阶段的研究已经逐步收敛到 `query-specific rubric-based LLM-as-a-Judge` 的鲁棒性分析。这种聚焦有利于形成清晰、扎实的核心贡献，但也需要在论文中说明其与“大语言模型动态鲁棒测评”总目标之间的关系，避免让课题显得过度收缩。

#### （2）自动生成 rubric 阶段对 prompt 设计较为敏感
在自动生成 rubric 的预实验中发现，生成结果会显著受到 prompt 写法的影响，不同 prompt 会导致 rubric 的数量、粒度、表述方式和重复模式产生明显差异。如果不先解决生成质量和去重质量问题，后续统计结论容易混入 prompt 设计本身带来的干扰。

#### （3）理论模型仍存在理想化假设
虽然当前已建立统一概率模型，并能较好解释多 prompt 与 query-level 聚合现象，但模型中关于 prompt 效应、解码噪声和条件独立性的假设仍偏理想化。后续需要结合更多实验现象，对模型做进一步修正，以保证理论解释与实际实验设置更加一致。

#### （4）动态适应性与 OOD 部分尚未充分展开
当前工作已从静态评测流程中找到了鲁棒性问题的核心来源，并开始向动态 rubric 生成扩展，但关于 OOD 测评、自适应评测组件和更广义的任务迁移验证仍处于后续阶段，尚未形成完整实验闭环。

### 2. 拟解决方案

#### （1）以“固定 rubric 条件下的鲁棒性分析”为核心主线，逐步外推到动态场景
后续论文组织上，将把当前已经较成熟的重复采样、criteria 聚合、prompt 聚合三部分作为核心主线，先完整回答“为什么 Judge 会波动、波动如何传递、如何被聚合机制削弱或修正”，再将自动生成 rubric 和更动态的测评扩展作为自然外推，形成由静到动的研究逻辑。

#### （2）继续完善自动生成 rubric 的 prompt engineering 与质检流程
在大规模开展自动生成 rubric 实验之前，先通过 case study 继续优化生成 prompt 和去重 prompt，并增加必要的人为检查，保证 rubric 的原子性、可验证性、忠实性与非重叠性。只有在生成质量较稳定后，再进行大规模统计分析，避免前端生成质量影响后端结论。

#### （3）结合实验结果修正和细化理论模型
继续在已有统一概率模型基础上，引入更贴合实验设定的条件分布解释，细化 prompt 效应与解码噪声之间的关系，并通过新增实验对理论结论进行验证，使理论分析既能解释现有现象，也能为后续 rubric 生成实验提供统一视角。

#### （4）在后续阶段补充动态扩展与适应性验证
后续工作将以自动生成 rubric 为切入点，进一步考察当评分标准本身成为变化来源时，测评稳定性与一致性如何变化。若时间允许，将尝试从 rubric 扩展、任务变化或未见样本角度补充更接近动态测评与适应性的问题验证，使课题与开题报告中的“可扩展、动态”目标更紧密对应。
