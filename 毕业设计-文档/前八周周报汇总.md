# 毕业设计前八周周报汇总

## 第1周

### 该周总结
本周主要围绕毕业设计题目开展前期资料学习与问题界定。我重点阅读了 LLM-as-a-Judge 方向的综述和代表性论文，了解该技术的基本流程、典型应用场景以及现有方法的主要分类方式，对单分制评估、pairwise 评估、rubric-based 评估等思路做了初步整理。通过阅读，我认识到该技术虽然能有效缓解人工评测成本高的问题，但在稳定性、主观性和输入敏感性方面仍存在明显不足。同时结合任务书要求，对课题目标进行了拆解，开始思考如何从评测方法本身切入，研究其鲁棒性问题。在实践上，本周完成了本地 Python 环境、基础依赖和项目目录的整理，为后续实验工作打下基础。

### 下一周计划
下周计划继续补充相关文献阅读，重点跟进本周阅读中注意到的 query-specific rubric-based 评测思路，检索并阅读这一方向的代表论文，分析其在开放任务评测中的优势与局限。同时准备对现有公开数据集和候选评测任务做进一步筛选，为后续从具体评测范式出发搭建实验流程做准备。

### 问题与建议
当前主要问题是课题范围较大，开题阶段涉及动态测评、OOD、鲁棒性归约等多个方向，若同时推进容易分散精力。建议前期先聚焦一个更可操作的核心切入点，以便尽快产出可验证的实验现象和初步结果，再在此基础上逐步扩展到更复杂的动态评测情形。

## 第2周

### 该周总结
本周继续进行文献调研，并将重点放在第一周关注到的 query-specific rubric-based 技术上。我检索并阅读了这一方向的代表论文，重点关注其如何针对开放式任务按 query 构造 rubric-list，以提高评测的可解释性和针对性。结合对数据集形式的观察，我进一步认识到，相比更粗粒度的整体打分方式，query-specific rubric-based 方法更适合研究开放任务中的评测稳定性问题。基于此标准，我搜索并最终选定RubricHub_v1作为数据集，进而对齐数据结构进行了解和分析。

### 下一周计划
下周计划开始围绕 query-specific rubric-based 场景搭建基础实验框架，包括数据集加载、样本查看、Judge 调用接口和评分结果保存格式设计。同时准备选取少量样本进行试运行，先观察同一输入在该评测范式下是否会出现明显波动，以便为后续研究重采样、聚合与稳定性分析提供依据。

### 问题与建议
目前的问题是文献中对“鲁棒性”的定义较多，有的强调抗扰动能力，有的强调重复性，有的强调与人工标注的一致性，概念边界还不够清楚。建议后续在自己的实验中尽早固定一套可执行的指标体系，例如分别衡量稳定性和与 Ground Truth 的一致性，这样能避免后面实验目标不断摇摆。

## 第3周

### 该周总结
本周开始搭建实验原型，完成了数据读取、样本抽取、生成模型输出保存以及 Judge 打分结果的基本存储结构设计，初步打通了从 query、response、rubric-list 到 score 的链路。我对评分 prompt 的构造方式进行了尝试，并开始关注不同 Judge 返回格式不统一的问题，思考后续如何将其规整为统一数值形式。通过少量样本的试运行，我已经观察到同一组输入在重复评审时会出现分数波动，这使得“单次 Judge 结果不是稳定点值”这一判断更有依据，也促使我进一步考虑后续引入重采样和 bootstrap 分析。

### 下一周计划
下周计划把实验流程进一步补全，重点实现评分清洗、非法结果过滤、多次采样保存以及基础统计计算模块。在此基础上，准备先围绕固定 prompt、固定 rubric 的条件，系统测量不同 Judge 在多次采样下的波动程度，争取形成“重复采样与稳定性”的实验结果和可视化图表。

### 问题与建议
目前实现层面的主要问题是不同模型输出格式差异较大，若不先统一解析规则，后续统计指标会受到较大干扰。建议优先把评分解析和数据清洗模块做扎实，哪怕前期样本规模稍小也没关系，因为这一部分会直接影响后续所有实验结论的可靠性。

## 第4周

### 该周总结
本周围绕“重复采样是否能提高鲁棒性”开展了第一轮系统实验。我完成了评分结果的统一提取逻辑，并使用 bootstrap 方式模拟在给定采样次数下均值的波动，分别统计了 criteria-level 和 query-level 的稳定性指标。初步结果表明，随着采样次数增加，Judge 分数的波动显著下降，不同 Judge 模型之间也存在明显稳定性差异。这一结果让我逐步确认：在固定 prompt 和固定 rubric 下，LLM Judge 的单次输出更适合被理解为一个分布样本，而不是确定分数。同时，我注意到 query-level 的总体相对波动通常小于单个 rubric 的相对波动，这为后续研究聚合效应提供了线索。

### 下一周计划
下周计划在已有稳定性实验基础上，进一步引入 Ground Truth，考察重复采样除了“变稳”之外是否也会“变准”。同时会继续整理和规范实验代码，把 notebook 中的核心步骤逐步沉淀为可复用模块，为之后加入更多变量控制和补充实验做好准备。

### 问题与建议
本周的主要问题是虽然看到了“重采样能降波动”的现象，但还不足以说明这种做法是否真正提升评测质量，因为系统可能只是“稳定地偏离真实值”。建议后续必须把稳定性分析和与 Ground Truth 的一致性分析结合起来，避免只得到片面的“更稳但不一定更准”的结论。

## 第5周

### 该周总结
本周重点研究了重复采样对评测一致性的影响。我将 Judge 在不同采样次数下的平均得分与 `ground_truth.json` 中的参考评分进行比较，使用 Spearman 相关系数从 criteria-level 和 query-level 两个层面衡量与 GT 的一致性。实验结果显示，重复采样通常不只会降低方差，也会提升与 GT 的排序一致性，这说明单次评分中的一部分误差确实是可通过平均削弱的随机噪声。同时，query-level 的一致性整体上高于 criteria-level，这与上周观察到的 query-level 更稳的现象相互印证。到这一步，我的研究主线开始从“发现波动”进一步推进到“分析波动如何影响最终评测质量”。

### 下一周计划
下周计划围绕 query-level 与 criteria-level 的差异做进一步分析，尝试从统计角度解释为什么多个 rubric 聚合后的总分反而可能更稳、也更接近 GT。同时准备补充 case study，对若干高波动样本进行人工检查，看看这类样本是否在任务性质或评分标准上有共同特征。

### 问题与建议
目前的问题是实验现象已经比较明显，但解释还停留在经验层面，例如“聚合后更稳”还缺少较严谨的统计说明。建议后续边做补充实验边推进理论化，把观察到的规律尽量转化为可以表达在论文中的公式、条件和更明确的机制解释。

## 第6周

### 该周总结
本周一方面对高波动样本进行了案例分析，另一方面开始加入更多影响因素的探索。我检查了若干在不同 Judge 下都较不稳定的对话，发现它们往往集中在建议型、边界模糊、要求过宽或对抗性较强的问题上，说明不稳定性并不是均匀分布的。与此同时，我开始统计 rubric 数量、总分、问题长度、回答长度、GT 归一化得分等特征与稳定性、一致性指标之间的关系，初步看到一些结构性相关趋势。更重要的是，本周我逐渐形成了一个更清晰的框架：研究对象不是单一分数，而是 score distribution；需要分别讨论解码噪声、criteria 聚合和更上游任务结构对最终结果的影响。

### 下一周计划
下周计划引入 prompt 扰动实验，考察语义等价但形式不同的 prompt 是否会改变 Judge 的评分分布，并分析多 prompt 条件下的采样结果是更稳还是更准。同时会继续整理目前的实验现象，尝试将“解码噪声”和“聚合效应”统一到一个概率模型中。

### 问题与建议
当前的问题是影响因素较多，若全部平行展开，容易让论文主线显得分散。建议在写作和实验组织上坚持“先主线、后扩展”的原则，先把最核心的三个问题（prompt的影响、rubric聚合的影响、解码随机性的影响）讲清楚，再把 case study 和 factor analysis 作为支撑证据放入后续部分，这样整体结构会更稳。

## 第7周

### 该周总结
本周主要完成了输入敏感性相关实验，并在观察到实验现象后推进了理论分析。我构造了多种 prompt variant，对比了单 prompt 与多 prompt aggregation 在鲁棒性和与 GT 一致性上的差异，发现多 prompt 条件下单次采样更分散，但随着采样次数增加，其一致性会逐渐追平甚至反超单 prompt。基于这一现象，我开始建立统一的概率模型，将 Judge 评分拆解为真实得分、prompt 层面的系统偏移和解码噪声三部分，并围绕两个核心现象展开推导：一是多 prompt 条件下出现的 bias-variance crossover，二是 query-level 相对 SEM 小于 criteria-level 的聚合效应。本周已经完成这两个现象的理论推导与证明，使实验结果与数学解释开始形成对应关系。

### 下一周计划
下周计划在已有理论框架基础上，进一步把研究视角延伸到 rubric 生成层面，重点讨论 query-specific rubric-list 的生成方案，并思考如果 rubric-list 本身发生变化，会怎样影响 query-level 的稳定性和一致性。同时准备结合具体样本开展 case study，对 rubric 生成与去重过程中的 prompt 做针对性调整，为后续预实验做准备。

### 问题与建议
本周的问题是虽然已经得到了较清晰的理论解释，但理论模型中仍包含一些理想化假设，例如 prompt 效应与解码噪声的关系、不同 prompt 分布之间的实际差异等，后续还需要结合实验继续修正。

## 第8周

### 该周总结
本周重点转向 rubric 生成方案的讨论与设计。我围绕 query-specific rubric-based 场景，开始思考如果 rubric-list 不再固定，而是由模型自动生成，那么 rubric 的内容质量、粒度和重复程度是否会进一步影响评测稳定性。为此，我对 rubric 生成 prompt 的写法进行了多轮讨论与修改，重点关注 rubric 的原子性、可验证性、忠实性和非重叠性。同时结合 case study，对若干具体 question 的生成结果进行人工检查，发现当前生成与去重流程中仍存在语义重复、表述不够原子化、过度具体化等问题，因此进一步针对这些问题调整了 rubric 生成 prompt 和去重 prompt。经过这一轮 prompt engineering，我对后续如何开展自动生成 rubric 的预实验形成了更明确的方案。

### 下一周计划
下周计划正式开展自动生成 rubric 相关预实验，包括对同一 question 多次生成 rubric-list、进行语义去重、构造 rubric 矩阵，并在 query-level 上计算 R-SEM 和 Spearman 等指标。希望借此验证：当 rubric-list 本身成为一个变化来源时，是否也会出现与多 prompt 类似的“更异质、更分散但可能更去偏”的现象。

### 问题与建议
目前的问题是 rubric 自动生成阶段受到 prompt 设计影响很大，不同写法会显著改变 rubric 的数量、粒度和重复模式，因此前期需要投入较多精力做人工检查和 prompt 调整。建议后续先把生成质量控制在较稳定水平，再开展大规模统计实验，这样能避免把 prompt 设计问题与后续结论混在一起。
