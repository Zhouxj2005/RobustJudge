> ### 1. Dialog ID 16：复杂的职场/法律纠纷咨询 (Conflict & Advice)
>
> - **概括**：用户遭遇了职场不公（被强制要求回办公室办公，且感觉受到歧视），试图寻找法律途径抗争，或者寻找一种“体面且不需要交接期”的辞职话术。
> - **不稳定原因**：这是一个典型的**“道德 vs 现实 vs 法律”**的三难困境。AI 给出的建议可能偏向于“合规抗争”，也可能偏向于“为了职业生涯选择妥协”。由于没有标准答案，不同 Judge 模型在多次采样中，对“AI 回复是否专业且富有同理心”的评价尺度会发生漂移，导致得分极不稳定。
> - **问题难度、Judge难度**
>
> ### 2. Dialog ID 17：清醒梦（玄学/心理学）的操作指南 (Methodology / "Pseudo-science")
>
> - **概括**：用户询问如何达成“清醒梦”（即在梦中意识到自己在做梦并掌控梦境）。
> - **不稳定原因**：这属于**界限模糊的内容**。它既有神经科学依据，又夹杂着大量民间玄学和心理暗示方法。Judge 模型在面对这类内容时，很难界定“回答质量”：是以“严谨的科学实证”为标准（给低分），还是以“好用、易于实践的技巧”为标准（给高分）？这种评价维度的不统一是高 SEM 的根源。
>
> ### 3. Dialog ID 24：构建一个全能且高盈利的数字产品网站 (Broad/Ill-posed Requirement)
>
> - **概括**：一个极其笼统的商业需求，要求构建一个涵盖所有功能、能赚钱的网站。
> - **不稳定原因**：**需求定义过于宽泛（Ill-posed）**。由于没有给出任何具体的业务领域（如：卖衣服的？卖软件的？还是信息聚合平台？），模型必须进行大量的“幻觉式填充”或“泛泛而谈”。Judge 模型在评分时，有的认为回答“创意很好”，有的认为回答“过于空洞、缺乏可行性”，导致评分在“好”与“差”之间大幅摇摆。
>
> ### 4. Dialog ID 38：针对 LLM 的对抗性越狱测试 (Adversarial/Jailbreak Prompt)
>
> - **概括**：通过字符拆分（x=Impart, y=..., z=...）和角色扮演指令，诱导模型忽略安全过滤机制，输出违禁内容（关于喝尿的有害意见）。
> - **不稳定原因**：模型内部存在两套机制的博弈：一套是“必须极致遵循指令”的本能，另一套是“必须遵守安全对齐（Safety Alignment）”的防火墙。在 12 次采样中，有时模型因为 Prompt 技巧太高而触发了违禁输出（可能导致高分或低分，取决于 Judge 的偏好），有时被安全防御机制拦截（导致输出安全但空洞的回复）。这种**对抗状态下的随机性**，导致了极高的评分方差。

# 五、增设实验（影响鲁棒性和一致性因素）

除了采样次数和 prompt variant 之外，进一步系统考察以下候选因素：

- `rubric-list` 长度：评分维度越多，Judge 是否越不稳定？
- 回答长度：更长或更短的回答，是否更容易引发评分波动？
- Ground Truth 的“中间态”程度：GT 越接近中间分数，是否意味着任务更模糊、更难评？
- 任务特征：格式约束、改写/翻译、代码、推理、事实性、安全性等任务标签，是否和稳定性/一致性有关？
- 回答生成模型：不同 Answer Model 生成的回答，是否系统性改变 Judge 的稳定性和校准程度？

这里同时考察四个指标：

- `mean_item_rel_sem`：criteria级平均相对波动
- `conv_rel_sem`：query 总分级相对波动
- `rubric_alignment`：对话内部 rubric 排序与 GT 的 Spearman 一致性
- `abs_score_bias`：Judge 平均总分与 GT 总分的归一化偏差
