# Query-specific Rubric-based LLM-as-a-Judge 鲁棒性研究梳理与总结

## 0. 背景知识
+ 什么是LLM-as-a-Judge：对于（用户questino，模型回答response）的样本对，回答质量有一个隐含的GT(score)。LLM-as-a-Judge**核心目的**就是设计一个pipeline，读入（question，response），输出Eval(score)，减小Var(Eval)和E(|Eval-GT|)
+ 什么是Rubric-based：大量question是开放式、没有可验证答案的，如：“帮我写一篇800字的小学参赛作文”。为了更准确地评价response质量，我们需要基于question要求（包含：直接要求、潜在要求）设计一套评价标准**rubric**，将单个主观判断变为**多个是否符合rubric**的可验证判断，从而提高Judge的可解释性和准确性。
+ 什么是Query-specific：不同question下判断response是否优质的标准是不同的。Query-specific就是基于question设计rubric-list
+ 什么是LLM-as-a-Judge的鲁棒性：鲁棒性即输入加入噪声时系统的输出稳定性。在当前语境下，即（question，response）相同时，由于一系列原因（rubric-list不同、weight不同、judge prompt不同、解码随机性）造成最终Eval Score的波动范围。

## 1. 研究出发点

我的研究关注的是：在 **query-specific、rubric-based 的 LLM-as-a-Judge** 评测中，为什么同一个被评对象（question,response）在rubric上多次评审会出现波动，以及这种波动会如何影响最终的 query 级评测结果。

传统上，我们常把 Judge 对一个 rubric 的打分看成一个确定值；但更合理的视角是：

- 对于单个 rubric，LLM Judge 的输出本质上是一个**概率分布**，
- 对于一个 query，其最终 eval score 是所有 rubric 分数的加权和，因此它本身也对应一个**概率分布**。
- 因而，query 级评测的准确性与稳定性，并不只取决于“平均分是多少”，还取决于：
  - 分布的均值 `mu`（决定相对 Ground Truth 的系统偏差 `bias`）
  - 分布的波动 `sigma`
  - 实际评测时的采样次数 `n`

从这个视角出发，整个评测 pipeline 中有多种因素都会改变 query 级 score distribution：

- rubric 的构造方式会改变分数空间的划分方式
- rubric 的权重会改变聚合后的分布形态
- prompt 的语义扰动会改变 Judge 的条件分布
- 有限次解码会引入采样噪声，进一步影响最终 score 的鲁棒性

因此，这项研究的核心问题可以表述为：

**如果把 LLM Judge 评分视为一个分布，那么 query 级评测结果的鲁棒性究竟由哪些因素决定？重复采样、prompt 扰动、rubric 设计和聚合方式分别在其中起什么作用？**

---

## 2. 研究总框架

1. 固定rubric权重w（rubric设计可认为统一与w的设计），$Y=\sum_i P(p=p_i) \sum_j w_j * Y_j(p=p_i)$
2. 之后分别从三条主线研究鲁棒性：
   - **解码噪声**：重复采样是否能让评分更稳、更优。
   - **criteria聚合**：Y(p=p_0)是否比Y_i(p=p_0)更稳、更优。
   - **prompt聚合**：Y是否比Y(p=p_0)更稳、更优。
3. 最后再追问：在固定权重w情况下，哪些数据因素、任务因素、生成模型因素、rubric 因素会系统性影响这些结论。
---

## 3. 当前工作在整体框架中的位置

基于第 2 节的统一视角，当前工作并不是直接同时研究所有因素，而是先固定一部分问题，再逐步向外展开。

当前的实际研究路径可以表述为：

1. 先固定 query-specific 的 rubric-list，并暂时固定其聚合权重 `w`。
2. 在这一前提下，先研究单个 Judge score 的随机性来自哪里。
3. 再研究从 `criteria-level` 到 `query-level` 的聚合，会如何改变最终 score distribution。
4. 之后再研究 prompt 扰动引入的 distribution change，以及 prompt aggregation 是否可能优于单一 prompt。
5. 最后再回头问：rubric 数量、rubric 结构、任务属性、response 属性、生成模型属性等上游因素，会如何系统性影响前面这些结论。

也就是说，当前研究的核心不是直接优化整个 pipeline，而是先把问题分成三个层次来理解：

- **解码层**：在固定 prompt、固定 rubric 下，Judge 单次打分的随机性如何影响最终结果。
- **criteria 聚合层**：多个 rubric score 聚合成 query-level score 时，噪声与偏差如何传递或抵消。
- **prompt 聚合层**：不同 prompt variant 改变 Judge 条件分布后，最终均值、方差和偏差如何变化。

围绕这个框架，当前工作已经形成了五块彼此衔接的内容：

1. 解码噪声：重复采样如何影响鲁棒性
2. 解码噪声：重复采样如何影响与 Ground Truth 的一致性
3. prompt aggregation：多 prompt 如何改变 bias-variance trade-off
4. case study 与 factor analysis：哪些样本、哪些因素会系统性影响上述现象
5. 理论化：如何把这些实验现象统一到一个偏差-方差-聚合框架中

## 4. 当前实验设置与指标定义

### 4.1 数据与模型设置

当前实验使用的数据与模型设置如下：

- 数据：`RubricHub_v1` 中前 100 条 query
- 回答生成模型：`qwen2.5-72b`、`gpt-oss-120b`、`qwen3-235b`
- Judge 模型：`deepseek-v3.2`、`qwen2.5-7b`、`qwen3-32b`、`gpt-4o`
- 评分对象：每个 query 对应的 query-specific rubric list

### 4.2 分析粒度

当前分析明确区分两个层面：

- `Criteria-level`：单个 rubric item 的打分分布
- `Query-level`：所有 rubric score 加权聚合后的 query 总分分布

这两个层面的差异非常重要，因为它们分别对应：

- 单个评测维度本身的稳定性与准确性
- 最终 eval score 的稳定性与准确性

### 4.3 结果指标

为了同时刻画“稳不稳”和“准不准”，当前工作主要使用两类指标。

第一类是**鲁棒性指标**，即 bootstrap 下均值的标准误 `SEM`。它刻画的是：如果在给定设置下只采样 `n` 次并取平均，那么最终 score 还会有多大的随机波动。

第二类是**一致性指标**，即与 Ground Truth 的 Spearman correlation。它刻画的是：在给定设置下，Judge 的评分排序是否更接近参考标准。

其中鲁棒性既看绝对 SEM，也看相对 SEM：

- `Criteria-level relative SEM = SEM / rubric points`
- `Query-level relative SEM = SEM / total query points`

相对 SEM 的意义在于：不同 rubric、不同 query 的分值尺度不同，只有做了归一化，才能比较不同对象之间的相对波动率。

### 4.4 当前研究中的“固定项”与“变化项”

为了让实验设计更清楚，可以把当前工作中各部分的固定项和变化项明确区分出来。

在主实验中通常固定的是：

- `(question, response)`
- 对应的 rubric-list
- rubric 聚合方式

而变化的是：

- 解码采样次数 `n`
- Judge 模型
- prompt variant
- 上游样本属性与任务属性

这样做的目的，是把“score distribution 的变化”拆解到具体来源上，而不是把所有不确定性混在一起。

## 5. 第一部分：解码噪声如何影响鲁棒性

### 5.1 实验原因 / 探究的问题

在固定 rubric 和固定 prompt 的情况下，最先要回答的问题是：

**Judge 的单次解码随机性到底有多大？如果重复采样并求均值，能否显著降低最终 eval score 的不稳定性？**

这是整个研究的第一步，因为只有先确认解码噪声确实存在、且可被平均削弱，后续再讨论 prompt aggregation 和 rubric aggregation 才有基础。

### 5.2 实验设计

- 使用 bootstrap 重采样模拟“从已有采样中抽取 `n` 次并取均值”的过程
- 对 `n = 1, 2, ..., 12` 分别计算：
  - `Criteria-level` 绝对 SEM
  - `Query-level` 绝对 SEM
  - `Criteria-level` 相对 SEM
  - `Query-level` 相对 SEM

### 5.3 实验结果

结果非常明确：**随着采样次数增加，所有 Judge 的评分波动都明显下降。**

### 5.4 结果分析

这部分实验说明了三件事。

第一，Judge 单次打分中确实存在显著的**解码噪声**。也就是说，即便输入的 `(question, response, rubric-list)` 完全相同，模型输出的 score 也不是稳定单点，而是带随机波动的样本。

第二，重复采样后取均值是有效的降噪手段。这个结果与“均值方差会随样本数增加而下降”的统计直觉一致，也为你后续把 Judge score 视为分布提供了经验支持。

第三，`Query-level` 往往比 `Criteria-level` 更稳，说明从多个 rubric score 到 query 总分的聚合并不是简单累积噪声，而更像是存在一定程度的**噪声抵消**。这个现象后来成为你理论化工作的关键切入口。

### 5.5 实验结论

- 在固定 prompt 和固定 rubric 下，Judge 单次评分具有明显随机性
- 重复采样显著提高了 `criteria-level` 和 `query-level` 的鲁棒性
- Judge 模型的选择本身就是重要的鲁棒性变量
- `query-level` 聚合显示出初步的方差抵消现象

### 5.6 下一步实验方向 / 内容

- 在“变稳”之外，进一步考察重复采样是否也会“变准”
- 理论解释为什么 `query-level` 会系统性地比 `criteria-level` 更稳
- 分析高波动样本是否具有结构性来源

## 6. 第二部分：解码噪声如何影响与 Ground Truth 的一致性

### 6.1 实验原因 / 探究的问题

只看鲁棒性是不够的。一个系统完全可能“稳定地偏离 GT”。因此第二部分要回答的是：

**在固定 prompt 和 rubric 的前提下，重复采样不仅是否能让 score 更稳，还是否能让 score 更接近 Ground Truth？**

### 6.2 实验设计

- 使用 `ground_truth.json` 作为参考标准
- 对 `n = 1, 2, ..., 12` 做 bootstrap 重采样并求平均分
- 分别计算：
  - `Criteria-level`：单个 query 内 rubric score 排序与 GT 的 Spearman
  - `Query-level`：跨 query 的归一化总分与 GT 总分的 Spearman

### 6.3 实验结果

整体趋势是：**随着采样次数增加，与 Ground Truth 的一致性普遍提升。**

### 6.4 结果分析

这一阶段说明，重复采样的作用不只是降低方差，而且在当前数据上通常还能带来更好的排序一致性。换句话说，单次打分里的误差有相当一部分确实是**可通过平均消除的噪声**，而不是全部都是不可修正的系统偏差。

另外，`Query-level` 的一致性普遍高于 `Criteria-level`，这与上一部分的观察是一致的：query aggregation 不仅减少了相对波动，也让最终总分的排序更接近 GT。

这说明 query-level aggregation 可能同时在做两件事：

- 降低随机波动
- 平滑局部 criteria 层面的判断偏差

### 6.5 实验结论

- 重复采样通常同时提升鲁棒性和与 GT 的一致性
- 更强的 Judge 往往表现为“又稳又准”
- `query-level` 聚合相比 `criteria-level` 更容易获得较好的 GT 对齐

### 6.6 下一步实验方向 / 内容

- 解释 query aggregation 为什么能同时提升“稳”和“准”
- 进一步区分一致性提升到底来自降噪，还是来自 bias cancellation
- 在 prompt 扰动存在时验证这一结论是否仍成立

## 8. 第四部分：prompt aggregation 如何改变 bias-variance trade-off

### 8.1 实验原因 / 探究的问题

前两部分主要研究的是固定 prompt 下的解码噪声，第三部分研究的是固定 prompt 下的 criteria aggregation。但在真实评测中，还有一个重要来源是：

**Judge 并不是只对 response 敏感，也会对 prompt 的表达形式敏感。**

因此这里要问的是：

1. 语义等价但形式不同的 prompt，是否会改变 Judge 的 score distribution？
2. 如果不只用一个 prompt，而是对多个 prompt variant 做 aggregation，最终是更稳还是更准？

### 8.2 实验设计

- 构造 4 类 prompt variant：
  - 翻译成英文
  - 翻译成西班牙语
  - 用 `deepseek-v3.2` 改写
  - 用 `qwen2.5-7b` 改写
- 当前重点分析的 Judge 为 `qwen3-32b`
- 将不同 variant 下的采样结果展平，视作更大的采样池
- 对比：
  - 单 prompt 条件
  - 多 prompt aggregation 条件

考察指标包括：

- `Criteria-level` / `Query-level` 鲁棒性
- 与 Ground Truth 的 Spearman 一致性

在鲁棒性上，多 prompt 并没有直接带来更小方差，说明多 prompt 的单次采样更“散”。

但在与 GT 的一致性上，多 prompt 并不简单变差，而是表现出“前期不占优、后期追平并可能反超”的现象。

### 8.4 结果分析

这个结果非常符合你的整体框架。

单一 prompt 可以理解为从某个固定条件分布中采样，它的优点是条件方差较小；但缺点是它可能带有这个 prompt 特有的系统偏差。

多 prompt aggregation 则相当于从多个条件分布中取样。这样做的代价是：

- 引入更大的分布异质性，导致方差上升

但它的潜在收益是：

- 平均掉单一 prompt 的特定偏差，使整体均值更接近更“本真”的评估中心

所以它不是“更稳”或“更不稳”的简单问题，而是一个典型的 bias-variance crossover 问题：

- 小样本时，方差主导，单 prompt 可能更好
- 大样本时，偏差主导，多 prompt 可能反超

### 8.5 实验结论

- Judge 对 prompt 语义等价改写具有显著敏感性
- prompt aggregation 的主要作用不是降方差，而更像是一种潜在的去偏机制
- 多 prompt 的价值更可能体现在高采样 regime，而不是低采样 regime

### 8.6 下一步实验方向 / 内容

- 形式化给出多 prompt 优于单 prompt 的 crossover 条件
- 研究固定预算下最优的 prompt 数量与每个 prompt 的采样次数分配
- 比较不同类型 prompt 扰动的去偏能力是否不同

## 9. 第五部分：高波动样本、任务结构与影响因素分析

### 9.1 实验原因 / 探究的问题

前三部分回答的是“解码、criteria aggregation、prompt aggregation”分别如何影响最终分布；但还需要继续问：

**这些现象在哪些样本上更强？哪些任务或数据属性会系统性放大不稳定性与偏差？**

### 9.2 实验设计

当前工作从两个方向补充这部分问题。

第一个方向是 case study：

- 提取不同 Judge 最不稳定的 top-k item / query
- 检查它们是否在不同 Judge 间重合
- 手动分析高波动典型样本，如 `Dialog 16/17/24/38`

第二个方向是 factor analysis：

- 数值因素：
  - `rubric_list_len`
  - `total_points`
  - `prompt_words`
  - `response_words`
  - `gt_norm_score`
  - `gt_midness`
- 结构因素：
  - 任务标签 / 主任务类型
  - 回答生成模型 `gen_model`

对应结果指标包括：

- `mean_item_rel_sem`
- `conv_rel_sem`
- `rubric_alignment`
- `abs_score_bias`

### 9.3 实验结果

当前 case study 的主要发现是：

- 高波动样本与 `rubric_list_len` 没有简单线性正相关
- 某些高波动样本在不同 Judge 间存在重合
- 高波动样本更多出现在以下类型：
  - 价值判断或建议型问题
  - 标准边界模糊的问题
  - 要求过宽、任务定义不足的问题
  - 对抗性或 jailbreak 风格问题

当前 factor analysis 的主要发现是：

- `rubric_list_len`、`total_points`、`response_words` 越大，`Criteria-level` 相对波动通常越大
- `gt_norm_score` 越高，鲁棒性通常越好，与 GT 的一致性也越高
- `gt_midness` 越高，鲁棒性越差，一致性也略差
- `prompt_words` 与 `abs_score_bias` 呈更明显的正相关

### 9.4 结果分析

这部分工作说明，不稳定性并不是均匀地分布在所有任务上的。

很多高波动样本并不是因为“长”或“复杂”，而是因为**判定标准本身缺乏刚性**。当 rubric 所要求的质量维度存在较大解释空间时，不同采样下的 Judge 更容易发生漂移。

同时，factor analysis 又进一步表明，上下文复杂度、分值结构和 GT 难度也会系统性地影响最终表现：

- rubric 更多、总分更大、response 更长时，criteria-level 更容易波动
- GT 越接近中间分，说明对象越边界化、越难判断，Judge 也越容易不稳定

因此，上游任务结构和 rubric 结构并不是背景条件，而是影响 score distribution 的核心变量。

### 9.5 实验结论

- 不稳定性具有明确的任务结构来源，而不是纯粹的模型随机性
- “难评”的 query 往往体现在标准模糊、边界样本和对抗性场景中
- factor analysis 已经把若干潜在机制从经验直觉推进到了可量化层面

### 9.6 下一步实验方向 / 内容

- 把“任务可判定性”进一步转化成更显式的可测变量
- 将 `rubric_alignment` 替换或补充为更直观的误差指标
- 通过控制实验验证 rubric 数量、区分度和 points 分布对鲁棒性的因果影响

## 10. 当前理论化工作：把实验现象统一到“偏差-方差-聚合”框架中

基于前面几部分结果，当前理论化工作的目标已经比较清楚：不是分别解释每个实验现象，而是把它们收束成一个统一视角。

### 10.1 对解码噪声的解释

在固定 prompt、固定 rubric 下，每次 Judge 打分都可以看作从某个条件分布中采样。于是重复采样后均值变稳，本质上就是：

- 单次 score 有随机噪声
- 均值化会降低噪声项的方差

这对应的是最基础的采样平均效应。

### 10.2 对 criteria aggregation 的解释

从多个 rubric 到 query-level 总分，是对多个随机变量的加权聚合。只要各 rubric 上的误差不是完全正相关，聚合后的相对波动就会被部分抵消。

因此：

- `Query-level R-SEM` 通常小于 `Criteria-level R-SEM`
- query-level 更稳不是经验巧合，而是 aggregation 结构决定的

### 10.3 对 prompt aggregation 的解释

prompt aggregation 的本质是把单一条件分布扩展为多个条件分布的混合。这样会同时带来：

- 更大的条件异质性，导致方差增大
- 更弱的固定 prompt 偏差，导致长期 bias 可能减小

于是自然出现：

- 小样本时方差主导，单 prompt 占优
- 大样本时偏差主导，多 prompt 反超

## 11. 当前研究的整体结论

基于目前的实验、分析和理论化工作，当前研究已经形成了一条比较完整的主线：

1. `LLM-as-a-Judge` 的输出应被理解为分布，而不是点值。
2. `Query-level eval score` 是多个 rubric-level score distribution 的加权聚合结果，因此它本身也是分布。
3. 在固定 prompt 和 rubric 下，Judge 单次打分存在显著解码噪声，重复采样能显著降低方差。
4. 重复采样不仅提升鲁棒性，通常也提升与 Ground Truth 的一致性。
5. `Criteria aggregation` 会带来方差抵消，因此 query-level 往往比 criteria-level 更稳、更接近 GT。
6. `Prompt aggregation` 带来的是 bias-variance trade-off，而不是简单意义上的“更稳”。
7. 最终鲁棒性并不只由 Judge 模型决定，还由任务可判定性、response 长度、GT 难度、rubric 结构等因素共同决定。

因此，这项研究的核心贡献不只是描述“Judge 会波动”，而是提出并逐步验证了一个更本质的判断：

**query-specific rubric-based LLM-as-a-Judge 的鲁棒性，本质上是一个由解码噪声、criteria aggregation、prompt aggregation 与上游 rubric 设计共同决定的统计问题。**

## 12. 当前最值得继续推进的问题

结合你现在的整体思路，后续最值得推进的方向已经比较明确：

- 更严格地完成多 prompt 的理论推导，尤其是 crossover 条件
- 系统研究 rubric 数量、权重、正交性与 query-level 鲁棒性的关系
- 从“query-level 为什么更稳”推进到“什么样的 rubric 结构最优”
- 直接生成 rubric，并检验“更多 rubric / 更可区分 rubric”对最终鲁棒性与一致性的影响
- 在固定预算下研究最优采样策略：多次单 prompt，还是少次多 prompt
- 将当前 factor analysis 推进到更接近因果验证的控制实验

从研究成熟度上看，你现在已经具备了：

- 一个清晰的问题定义
- 一个统一的 score distribution 视角
- 一组能相互支撑的实验事实
- 一个正在成形的理论解释框架

下一步最关键的工作，不是再继续平铺更多实验，而是把这些实验进一步收束成一个更紧的理论叙事，并把它自然推进到 rubric design 和 sampling strategy 的优化问题上。
